# 03 - Hourly Study Plan (Absolute Beginner)

Use this template each day.

## Daily 3-hour block template

1. 30 min - Read and understand concepts
2. 90 min - Implement code from day guide
3. 30 min - Test endpoints and debug
4. 30 min - Write summary notes and commit

## Rules for learning speed

1. If a concept is unclear, pause and search only that concept.
2. Do not copy large code blindly. Type it manually once.
3. Keep one terminal for server, one for commands.
4. Commit after each working milestone.

## Suggested commit messages by day

1. `day1: setup backend_v2 and db foundation`
2. `day2: implement users layered crud`
3. `day3: add jwt auth and role checks`
4. `day4: add document upload and persistence`
5. `day5: integrate verification service layer`
6. `day6: add background processing flow`
7. `day7: harden api with errors and logging`

## End-of-day reflection questions

1. What broke first and why?
2. Which layer did I accidentally violate?
3. Which part of code is still too large or unclear?
4. What should I refactor tomorrow before adding features?
